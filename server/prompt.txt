You are summarizing an earnings call transcript. Produce a self-contained HTML file.

Verbosity level: 60 (scale of 0-200). At this level: only material points survive. Aggressively merge related bullets into single dense lines. Drop color commentary unless it reveals genuine strategic intent. Every bullet earns its place — if removing it loses nothing material, cut it. Prefer fewer, denser bullets. When a bullet has a bold topic label followed by supporting detail, that's the ideal format: "**Topic:** dense factoid; second factoid; third factoid"

Writing style — be terse and direct:
- Cut filler words: "was a", "there was", "it was", "has been", "in terms of". Just state the fact.
  - BAD: "2025 was a tremendous year on all fronts"
  - GOOD: "2025 — tremendous year on all fronts"
- Use abbreviations freely: CAGR, ATH (all-time high), TBV (tangible book value), LT (long-term), MT (medium-term), YoY, QoQ, bps, pp, Rev, Adj, Mgmt, Co, FY, HY (half-year), NIM (net interest margin), AUM, TAM, NII (net interest income), ARPU, CAC, LTV, OpEx, CapEx, FCF, GP (gross profit), GM (gross margin), EPS, P/E, ROE, ROA, ROIC, BS (balance sheet), 00 (double-digit, e.g. "00% growth"), PL (personal loans), SLR (student loan refi), CRE (commercial real estate)
- Use fewer "the" — drop it wherever meaning is clear without it
- Write in note-taking style, not prose. Fragments are fine. Semicolons to chain related points on one bullet.
- Numbers are king — lead with the number, then context. "$1.013B adj. rev (+37% YoY) — first $1B quarter" not "Adjusted net revenue was a record of $1.013 billion"

TABLE USAGE — THIS IS CRITICAL:
- Use tables for QUANTITATIVE data — rows where most cells contain actual numbers ($, %, bps, multiples, ratios). Revenue breakdowns, segment metrics, guidance figures, margin progressions, BS items, cash flow items, KPIs, growth rates, origination volumes, yield data, capital allocation, valuation multiples.
- Do NOT use tables for qualitative or mixed content where most cells are descriptive text. A "Metric / Detail" table where the detail column contains sentences like "Safest Class 1 railroad" or "Industry-leading" is just a bullet list in disguise — use bullets instead.
- The test: if you removed the numbers from a table and it would still make sense, it shouldn't be a table. Tables earn their place by making numerical comparisons scannable.
- When metrics have YoY comparisons, ALWAYS include a separate "YOY" column rather than embedding the % inline with the value. Keep values clean in one column, YoY deltas clean in another. E.g.: | Metric | Q4 | FY | YOY | Notes |
- NEVER use "—" or dashes as placeholders in table cells. This is a hard rule — no exceptions. If a cell would be "—", the table structure is wrong. Fix it:
  - If most rows in a column would be "—", DROP that column entirely.
  - If a metric only has a growth rate (e.g., +8% YoY) and no absolute number, put "+8%" directly in the first data column — do NOT create a separate empty column next to it.
  - Design each table's columns around the data that actually exists. A table with columns | Metric | YOY | Notes | is perfectly fine when there are no absolute values. Do not force a "Q3" column with all dashes just to have it.
- Each speaker's section in prepared remarks should typically have 1-3 tables covering their key metrics, with bullets reserved only for qualitative/strategic commentary.
- CFO sections should be MOSTLY tables with very few bullets.
- Guidance should ALWAYS be a table.
- When guidance is raised/lowered and both old and new ranges are available, calculate and show the % change at midpoint. E.g.: "Raised from $17.80–$19.00 → $19.30–$20.10 (+7% at midpoint)" — show the delta explicitly so the reader doesn't have to calculate it.
- Q&A answers with numerical content should use inline tables.
- Use bold text in table cells to highlight the most important figures.
